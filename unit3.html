Operating System - Introduction

An operating system (OS) is an integrated set of programs that controls overall resources such as CPU, memory, and input-output devices of the computer system. The major objective of the operating system is to improve the performance and efficiency of a computer system. Like the manager of a company, an operating system is responsible for the smooth and efficient functioning of the entire computer system. The operating system provides the platform for other application programs/software to run and execute. It provides users with an interface so that users can easily communicate with computers, which is more convenient to use and operate.

An operating system is a collection of programs that controls the overall operation of the computer system. It also controls and coordinates the use of hardware among various application programs for various users. So, it acts as an interface between users and computer hardware. The other major functions of an operating system are:

  • It controls, monitors, and coordinates the overall operation of the computer system.
  • It acts as an interface between the user and the computer.
  • It provides a platform to develop, run, and execute other computer programs.
  • It manages hardware resources such as CPU, memories, input-output terminals, networking equipment, etc.
  • It hides programming and hardware complexity from the user.

The primary goal of an operating system is to maximize the productivity of a computer system by operating it in the most efficient manner while minimizing human intervention. Some examples of operating systems are MS-DOS, Windows, Linux, MacOS, UNIX, Chrome, Fedora, Ubuntu, etc.

Functions / Features / Advantages of Operating System (OS)

  1. Input-Output (I/O) Management: Allows computer to interact with peripheral devices like keyboard, mouse, printer, scanner, etc.
  2. User Interface (UI): Provides an ideal environment where users can interact with the computer system. There are two types of user interfaces:
    • Character/Command User Interface (CUI): E.g., MS-DOS
    • Graphical User Interface (GUI): E.g., Windows, MacOS
  3. Security: Protects against threats such as viruses, unauthorized access, etc., through built-in tools like user authentication and password protection.
  4. Process Management: Allocates the CPU to execute a chosen process, managing job scheduling, process scheduling, and dispatching.
  5. Memory Management: Allocates memory to load programs and data into RAM and frees up memory when the program terminates.
  6. Data Management: Organizes data into logical groupings called files, allowing efficient storage and retrieval.
  7. Command Interpreter: Interprets user commands and translates them into actions the computer can execute.
  8. Dead-Lock Prevention: Ensures that resources are allocated properly to prevent deadlock situations in processing.
  9. Time Sharing: Allows the CPU to allocate time among multiple users on the same computer system.
  10. Virtual Memory: Compensates for a shortage of physical memory by using space on secondary storage like hard drives.

Types of Operating System (OS)

A) On the basis of Processing

  • Multi-programming OS: Executes multiple user programs simultaneously using a single processor.
  • Multi-tasking OS: Runs multiple programs concurrently in a single-user system.
  • Multiprocessing OS: Uses multiple processors to run programs simultaneously, improving execution speed.
  • Time-Sharing OS: Allows multiple users to share CPU time and provides quick responses.
  • Batch Processing OS: Groups similar jobs together to be executed in sequence.
  • Multi-threading OS: Divides a process into smaller threads that can be executed concurrently.
  • Real-Time OS: Provides quick processing for critical tasks, ensuring tasks complete in a predefined time.
  • Online Processing OS: Processes transactions as they happen in real-time, allowing for user interaction during processing.

B) On the basis of User Interface (UI)

  • Character User Interface (CUI): Involves interaction with the system through commands typed by the user (e.g., MS-DOS, UNIX).
  • Graphical User Interface (GUI): Involves interaction through graphical elements like icons, windows, and menus (e.g., Windows, MacOS).

C) On the mode of user

  • Single User: OS developed for a single user at a time (e.g., MS-DOS).
  • Multi User: OS developed for multiple users to access a single system at the same time (e.g., Linux, UNIX).

Some important Operating Systems in use

  • Open Source Operating System: Source code is freely available for modification and redistribution (e.g., Linux, UNIX).
  • UNIX: A powerful multi-user, multi-tasking OS developed by AT&T Bell Labs in the 1970s.
  • Linux: An open-source OS based on UNIX, widely used for server systems and available in different distributions (e.g., Red Hat, Ubuntu).
  • MacOS: A GUI-based OS used in Apple computers with high-quality graphical interface.

Some Terminologies related to OS

  • Spooling: A process to handle the difference in speed between CPU and slow peripheral devices like printers.
  • Process: A program in execution. It can be in a running, ready, or blocked state.
  • Thread: A smaller sub-process of a larger program that can run concurrently with others.
  • Scheduling: The OS's process of deciding which process to run next (e.g., priority scheduling, time-sharing scheduling).
  • Buffering: Using a small, high-speed storage device to hold data during I/O operations.
  • Virtual Memory: Uses secondary storage to supplement primary memory when running multiple programs simultaneously.
